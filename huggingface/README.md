## Documentation Boundary

The Hugging Face cards are public-facing release documents. They should include
model purpose, intended use, limitations, license boundary, privacy boundary,
website links, and repository links. They should not include:

- private hostnames, IP addresses, ports, or local paths;
- exact internal GPU topology or fleet inventory;
- failure runbooks or recovery recipes;
- raw benchmark rows that reveal deployment capacity;
- client material, privileged work product, or private prompts;
- secrets, credentials, or tokens.
